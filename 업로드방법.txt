================================================
GitHub 업로드 방법 (폴더 구조 유지가 핵심!)
================================================

지금 문제: 파일이 전부 루트에 평평하게 올라가서 404.
해결: api/ public/ 폴더 구조를 그대로 유지해서 올려야 함.

────────────────────────────────────────
■ 방법 A: 기존 레포 파일 싹 지우고 다시 올리기 (권장)
────────────────────────────────────────
1. 이 zip을 PC에서 압축 해제
   → addr-refine-deploy 폴더가 생기고, 그 안에 api/, public/ 등이 있음

2. GitHub 레포(newwonwoo/unisquads)에서 기존 파일 전부 삭제
   (또는 새 레포를 만들어도 됨)

3. GitHub 레포 → "Add file" → "Upload files" 클릭

4. ★핵심★ 압축 푼 addr-refine-deploy 폴더를 열고,
   그 "안의 내용물"(api 폴더, public 폴더, vercel.json,
   requirements.txt, README.md)을 통째로 선택해서
   업로드 창에 드래그&드롭.

   ※ 파일 하나씩 열어서 올리지 말 것! 폴더째 드래그해야
     api/juso.py 같은 구조가 유지됨.
   ※ 크롬에서는 폴더 드래그가 됩니다. 폴더가 그대로 올라감.

5. Commit changes → Vercel이 자동 재배포

────────────────────────────────────────
■ 방법 B: 폴더가 드래그 안 되면 (파일 하나씩)
────────────────────────────────────────
GitHub → Add file → Create new file →
파일명 칸에 경로까지 입력:
   api/resolve.py     (api/ 붙이면 폴더 자동생성)
   api/juso.py
   api/kakao.py
   api/iros_api.py
   api/iros_resolver.py
   public/index.html
   public/app.js
각 파일 내용을 복붙 → Commit. (번거로우니 방법 A 우선)

────────────────────────────────────────
■ 업로드 후 확인
────────────────────────────────────────
GitHub 레포에서 구조가 이렇게 보여야 정상:
   📁 api        ← 폴더!
   📁 public     ← 폴더!
   📄 vercel.json
   📄 requirements.txt

api, public이 "폴더"로 보이면 성공.
파일이 평평하게 흩어져 있으면 실패(다시).
