# unisquads 주소정제·IROS 품질확인서

확인일: 2026-07-19
대상: Vercel `main` 배포용 소스

## 판정

정적·회귀 테스트 기준 적합. 실 IROS 서비스 시간/세션에 의존하는 외부 통합 호출은 배포 URL에서 별도 확인 대상이다.

## 확인 항목

| 항목 | 코드상 확인 결과 |
|---|---|
| Vercel 설정 | 유효 JSON, `public` 출력, 서울 리전 `icn1` |
| API import | `api/resolve.py`가 `resolve_one_api`를 직접 import하고 import 실패 원인을 응답에 노출 |
| 주소 원천 | JUSO로 주소/PNU 확정, 네이버는 건물명 복구 후 JUSO 재확인 |
| 주소 배치 | 고유 원문주소 호출, 주소군 보강, 행별 결과 객체 분리 |
| 멱등성 | 입력·모듈·상위근거·결과 지문 검증 후 행별 재사용 |
| 옛주소 | 명시적 방향성 규칙만 허용, 규칙 ID/버전 기록 |
| IROS 배치 | PNU 단위 1회 완전수집 후 세대별 로컬 매칭 |
| 대단지 | `pageUnit=1000`, 필요시 순차 페이지 수집, 시간예산 초과 시 보류 |
| 완전성 | 총건수=원본수신수, 파싱오류 0, 정확지번 범위, 해시/스키마 지문 필수 |
| 세대 없음 | 완전수집 전에는 `REG_UNIT_NOT_FOUND` 확정 금지 |
| 복수지번 | 대표지번 실패 시 같은 법정동의 원문 대체지번을 주소군 1회 재수집 |
| 후보 정규화 | 복합동+호 접두, A/B 한글표기, `00동`, 하이픈 호를 보수적으로 처리 |
| 정확지번 | `add_item/sojae/lot_no`로 인접 부번 후보를 매칭 전에 제외 |
| 집합건물 호 누락 | IROS 호출 전 `UNIT_INPUT_REQUIRED` 분리 |
| 검토 플래그 | 지번범위·부동산구분·동호·건물명 근거가 모두 맞아야 자동확정 |
| 감사 추적 | 주소/IROS 상태, 지문, 버전, 수집수, 페이지, 해시, 매칭근거를 엑셀 출력 |

## 자동 검증

- 브라우저 모듈 구문 및 링크 테스트
- 주소 파이프라인 지문 재사용/무효화/객체 격리 테스트
- 주소군 전파 2회 실행 동일성 및 기준 PNU 변경 무효화 테스트
- 옛주소 방향성·지역범위·임의 접미변경 차단 테스트
- IROS pageUnit 협상·완전성·부분응답·파싱누락·부동산구분·동호 매칭 테스트
- JavaScript/Python 공통 동호 벡터 테스트
- 대체지번·복합동·동 공란·건물명 교차검증·인접 부번 회귀 테스트

실행 명령:

```bash
node --check public/app.js
node --test tests/*.mjs
python -m unittest discover -s tests -p 'test_*.py'
```

## 배포 후 최소 확인

1. `/api/resolve?addr=서초구%20서초동%20967`이 import 오류 없이 IROS 상태를 반환하는지 확인
2. 주소정제 배치 1회 후 같은 파일을 다시 실행했을 때 재사용 행 수와 결과지문이 유지되는지 확인
3. 동일 PNU의 서로 다른 동·호가 한 번의 후보수집 결과에서 각각 올바른 고유번호로 매칭되는지 확인
4. 부분수집 또는 서비스시간 외 응답이 `REG_UNIT_NOT_FOUND`로 바뀌지 않는지 확인

외부 API 환경변수는 `JUSO_CONFM_KEY`, `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET`을 사용한다.
