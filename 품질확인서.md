# addr-refine Vercel 완전 올인원 v1.0 품질확인서

작성일: 2026-07-08 | 판정: **적합 (배포 준비 완료 / IROS 세션은 배포 후 URL에서 1회 확인)**

## 1. 요구사항 적합성
**요구**: 로컬 파이썬/배시 없이 웹으로 배포해서 쓰는 완전 올인원

| 요구 | 구현 | 판정 |
|---|---|---|
| 로컬 실행 제거 | Vercel 서버리스에 전부. GitHub 푸시→자동배포. 로컬 아무것도 실행 안 함 | ✅ |
| 정제 URL 하나 | public/ 웹앱 + api/juso·api/kakao 프록시(키 서버보관) | ✅ |
| 등기 URL 하나 | api/resolve.py(iros_api 직결) → 같은 도메인 /api/resolve | ✅ |
| 배포=테스트 | 세션 확인을 배포된 URL에서. 로컬 삽질 불필요 | ✅ |
| 폰 사용 | 반응형 웹앱, 어디서나 URL 접속 | ✅ |

## 2. 구조
```
public/index.html, app.js       프론트(정제 UI + 등기조회, 프록시 연결)
api/resolve.py                  등기고유번호 (iros_api)
api/juso.py, api/kakao.py       정제 프록시 (환경변수 키)
api/iros_api.py, iros_resolver.py  리졸버 코어
vercel.json, requirements.txt   런타임·라우팅
```

## 3. Vercel 적응 처리
- **프록시화**: makeRealClients가 juso/카카오 직접호출 → /api/juso·/api/kakao 경유(CORS·키노출 해결)
- **기본 모드 real**: 배포판은 목업 아닌 실제 프록시 기본
- **브리지 /api**: 등기 조회가 같은 도메인 /api/resolve 기본 사용
- **헬스체크 분기**: /api 모드는 서버리스 상시가용이라 항상 연결됨 처리
- **storage 심**: 실브라우저 localStorage로 .env 저장(설정 유지)

## 4. 4단계 검증
1. 문법: 파이썬 함수 5개 전부 ast 통과
2. pyflakes: clean
3. 로직:
   - 임포트 체인(resolve_one_api·normalize_sido) 정상
   - payload 실측일치(swrd=서초구 서초동 967, 시도=서울특별시)
   - 실 응답구조 파싱(dataList+pin_land 18→14) 정상
   - 프론트 번들 구문검증 통과
4. vercel.json 유효, handler 클래스 3함수 확인

## 5. 잔여 검증 (배포 후 URL에서 1회)
- **정제**: 환경변수(JUSO_CONFM_KEY·KAKAO_REST_KEY) 설정 후 주소 입력 → PNU 표시 확인
- **등기 세션**: "등기고유번호 조회" → RESOLVED(1102-1996-101140 등) 확인
  - 성공 → 완전 올인원 종결
  - ERROR → IROS 세션이 서버리스 requests로 안 잡히는 것 → EC2 브리지 폴백
    (웹앱 ⚙ 설정 BRIDGE_URL에 EC2 주소 입력, iros-resolver 패키지 활용)
